# 当前功能地图

本文档按当前管理后台的信息架构整理 TokenHub 已成型的功能域，用于对齐产品说明、接口文档、验收和后续开发优先级。

## 功能域总览

| 功能域 | 页面/入口 | 核心能力 |
| --- | --- | --- |
| 总览 | 网关概览、模型演练场、接口文档 | 查看整体运行状态，测试模型链路，向业务开发者说明模型 API 调用方式 |
| AI 接入 | Provider 渠道、模型目录、路由策略 | 管理上游渠道、统一模型目录、模型到 Provider 的候选路由和优先级 |
| 企业治理 | 项目空间、API Key、团队分组、用户管理、审批记录 | 管理项目、Key、团队、成员、角色和额度提升审批 |
| 成本审计 | 用量统计、请求日志、成本账单、成本中心、导出报表 | 查看请求、Token、成本、成员/项目/Provider 维度统计和报表导出 |
| 健康与告警 | 健康检测、告警规则、告警事件、通知渠道、通知记录 | 默认检测 Provider 与额度风险，触发告警并通过配置渠道通知 |
| 安全运维 | 安全策略、代理出口、数据备份、公告通知、系统设置 | 管理脱敏/IP/错误透传策略、出口代理、SQLite 备份、公告和基础配置 |

## 角色与数据范围

| 角色 | 主要职责 | 默认可见范围 |
| --- | --- | --- |
| 管理员 | 全局配置、Provider、模型、路由、用户、备份和系统设置 | 全局 |
| 安全管理员 | 请求日志、安全策略、告警、通知和审计事件 | 全局安全域 |
| 团队负责人 | 团队成员、团队项目、团队用量和团队成本 | 所属团队 |
| 普通用户 | 创建自己的 Key，查看自己的用量和请求日志 | 自己 |

权限边界必须由后端 Admin API 和模型 API 共同执行。前端菜单隐藏只用于降低噪音，不能作为安全边界。

## 当前闭环

### 模型调用闭环

1. 业务应用使用 TokenHub API Key 调用 `/v1/*`。
2. 网关校验 Key、项目状态、模型白名单、额度和并发。
3. 路由引擎按统一模型名找到候选 Provider。
4. 按优先级、权重、策略和健康状态选择上游。
5. Provider Adapter 转发请求并归一化响应。
6. 记录请求日志、路由尝试、用量、成本和告警事件。

### 项目额度闭环

1. 管理员在项目空间中配置项目额度。
2. 请求触发请求数、Token、成本或并发限制时，网关返回清晰错误。
3. 项目页面出现额度提升申请入口。
4. 申请进入审批记录。
5. 管理员或负责人批准后更新项目额度。

### Provider 路由闭环

1. 管理员新增 Provider，并测试连接。
2. 从 Provider 模板选择上游模型。
3. 将上游模型映射到统一模型目录。
4. 一个统一模型下维护多个 Provider 候选。
5. 管理员通过拖拽顺序、优先级和权重调整默认调用顺序。
6. 健康检测失败时自动生成告警，路由跳过不可用候选。

### 请求日志闭环

1. 每次模型调用都有 request_id。
2. 日志记录项目、Key、模型、Provider、路由、状态、延迟、Token 和成本。
3. 详情页展示 request/response 的脱敏内容或摘要。
4. 普通用户只能看自己的日志；团队负责人看团队日志；安全管理员和管理员可看全局日志。
5. 导出受权限控制，并记录管理操作审计。

### 健康告警闭环

1. 系统默认内置 Provider 健康和额度风险规则。
2. 健康检测可手动运行，也应支持定时巡检。
3. 告警事件记录触发对象、级别、原因和时间。
4. 通知渠道支持 Webhook、飞书、钉钉、企业微信、Slack 和邮件。
5. 通知记录记录发送目标、状态码、失败原因和重试结果。

## 当前不作为主线的能力

- 支付系统、自助充值和面向个人用户的套餐售卖。
- 复杂账号代理网络和规避上游规则的能力。
- 复杂财务分摊、内部发票和部门账单工作流。
- 多区域高可用集群调度。
- 模型质量自动评分。
- 完整工单系统。

## 后续增强优先级

| 优先级 | 功能 | 说明 |
| --- | --- | --- |
| P0 | 后端 RBAC 数据范围校验 | 普通用户、团队负责人、安全管理员、管理员的 API 返回范围必须隔离 |
| P0 | Provider 凭证加密 | Provider API Key、SMTP 密码、身份源密钥需要落库加密和不回显 |
| P0 | 请求日志脱敏与保留策略 | 明确 request/response 保存、截断、脱敏、保留周期和导出权限 |
| P1 | OIDC/OAuth/LDAP 登录链路 | 身份源配置已具备，下一步接入登录回调、用户同步和角色映射 |
| P1 | 定时健康巡检 | Provider、路由和额度风险默认自动检测 |
| P1 | SQLite 定时备份与恢复演练 | 手动备份已有，下一步补定时任务、校验和恢复演练 |
| P2 | OpenAPI 导出 | 生成管理 API 和模型 API 的 OpenAPI 文档 |
| P2 | 生产 Helm Chart | 为内网 Kubernetes 部署提供 values、持久卷、Ingress 和观测配置 |
