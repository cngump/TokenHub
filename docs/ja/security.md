# セキュリティ

Language: [English](../security.md) | [简体中文](../zh-CN/security.md) | 日本語

TokenHub は、企業が管理する認可済みモデル API アクセスのためのゲートウェイとしてデプロイしてください。

## API Key

- API Key はプロジェクト内で作成します。
- Key は有効化、無効化、ローテーション、失効、有効期限、モデル許可リストに対応します。
- Key にはリクエスト数、Token、コスト、並行数の上限を設定できます。
- アプリケーションには必要なプロジェクトとモデルの権限だけを付与してください。

## 管理アクセス

- Admin API は Admin Token と管理セッションで保護されます。
- 本番利用前に開発用 Token を必ず変更してください。
- 管理コンソールはネットワーク、SSO、VPN、リバースプロキシでアクセス制限することを推奨します。

## ロールとデータ範囲

TokenHub はロールベースの UI とバックエンドデータフィルタリングを提供します。

- 一般ユーザーは自分の Key を作成・管理し、自分の利用量とリクエストログを確認できます。
- チームリーダーはチームメンバーとチーム利用量を確認できます。
- 管理者はグローバル設定を管理できます。
- セキュリティ管理者はセキュリティ関連ログ、アラート、ポリシーを確認できます。

フロントエンドのメニュー非表示は使いやすさのためのものです。データ範囲はバックエンド API で必ず制御してください。

## Provider 認証情報

- 企業所有で正式に認可された Provider 認証情報を使用してください。
- Provider API Key をフロントエンドコードや公開ドキュメントに露出しないでください。
- Provider Key は社内ポリシーに従ってローテーションしてください。
- 本番環境ではシークレット管理または保護された環境変数を推奨します。

## リクエストログ

リクエストログはトラブルシューティングと監査に有用です。本番環境でリクエスト/レスポンス本文を保持する前に、保持期間とマスキング方針を確認してください。

推奨制御:

- シークレットと機密フィールドをマスクします。
- リクエスト本文の閲覧を許可ロールに限定します。
- リクエスト本文とレスポンス本文に保持期間を設定します。
- 承認済みストレージにのみエクスポートします。

## ネットワーク制御

- 本番トラフィックは HTTPS で提供します。
- 可能な場合は Admin API へのアクセスをネットワーク単位で制限します。
- 企業ポリシーで統一出口が必要な場合は、上流 Provider 通信を承認済みプロキシ経由に設定します。
