# 安全

Language: [English](../security.md) | 简体中文 | [日本語](../ja/security.md)

TokenHub 应作为企业自控的模型 API 访问网关使用，面向已授权的模型服务访问场景。

## API Key

- API Key 在项目空间中创建。
- Key 支持启用、禁用、轮换、吊销、过期和模型白名单。
- Key 可配置请求数、Token、成本和并发限制。
- 业务应用只应获得完成自身任务所需的项目和模型权限。

## 管理访问

- Admin API 受 Admin Token 和后台会话保护。
- 生产环境必须替换开发 Token。
- 建议通过内网、SSO、VPN 或反向代理限制管理后台访问范围。

## 角色和数据范围

TokenHub 支持基于角色的菜单和后端数据过滤：

- 普通用户可以创建和管理自己的 Key，并查看自己的用量和请求日志。
- 团队负责人可以查看团队成员和团队用量。
- 管理员可以管理全局配置。
- 安全管理员可以查看安全相关日志、告警和策略。

前端隐藏菜单只用于降低界面噪音，真正的数据范围必须由后端 API 控制。

## Provider 凭证

- 使用企业自有、正式授权的 Provider 凭证。
- 不要把 Provider API Key 暴露在前端代码或公开文档中。
- 按企业内部策略轮换 Provider Key。
- 生产环境优先使用密钥管理系统或受保护的环境变量。

## 请求日志

请求日志可用于排障和审计。生产环境启用请求/响应内容留存前，请确认保留周期和脱敏策略。

建议控制：

- 对密钥和敏感字段脱敏。
- 仅授权角色可查看请求内容。
- 为请求体和响应体设置保留周期。
- 只导出到经过批准的存储位置。

## 网络控制

- 生产流量使用 HTTPS。
- 尽可能按网络边界限制 Admin API 访问。
- 如果企业要求统一出口，可将上游 Provider 流量配置到批准的代理出口。
