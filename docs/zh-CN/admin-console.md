# 管理后台

Language: [English](../admin-console.md) | 简体中文 | [日本語](../ja/admin-console.md)

管理后台是 TokenHub 的主要配置入口，默认地址为 `http://localhost:3000`。

## 主要区域

| 区域 | 作用 |
| --- | --- |
| 网关概览 | 查看运行状态、请求量、成本、Provider 状态和公告 |
| 模型演练场 | 按当前路由策略测试统一模型 |
| 接口文档 | 给业务开发者展示模型 API 调用方式 |
| Provider 渠道 | 配置上游 Provider、Base URL、API Key、模板和健康状态 |
| 模型目录 | 维护 TokenHub 对外暴露的统一模型名 |
| 路由策略 | 将统一模型映射到 Provider 模型，并调整 Provider 顺序 |
| 项目空间和 API Key | 发放有权限、额度、模型白名单的项目 Key |
| 团队分组和用户管理 | 管理用户、团队成员和角色 |
| 用量统计和请求日志 | 查看 Token、成本、状态、延迟、请求内容和路由尝试 |
| 健康与告警 | 检查 Provider 健康，配置告警规则和通知渠道 |
| 数据备份和系统设置 | 管理 SQLite 备份、角色展示、身份源和系统配置 |

## Provider 配置流程

1. 打开 `Provider 渠道`。
2. 新建 Provider，填写类型、Base URL 和 API Key。
3. 测试连接。
4. 从 Provider 模板中选择上游模型。
5. 到 `路由策略` 中确认路由已创建。

## 路由配置流程

路由策略按统一模型展示。一个统一模型下面可以有多个 Provider 行。

- 拖动 Provider 行可以调整调用顺序。
- 第一条启用线路是主线路。
- 不健康的 Provider 或禁用路由会被跳过。
- 非流式请求在遇到可重试错误时，可以自动切换到下一条线路。

## Key 配置流程

1. 打开 `API Key`。
2. 在某个项目下创建 Key。
3. 如果需要限制模型，配置模型白名单。
4. 按需配置日额度、月额度、成本、Token 和并发限制。
5. 将生成的 Key 配置到业务应用或 SDK 测试脚本中。

## 请求日志

`请求日志` 会记录模型调用、选中的 Provider、上游模型、延迟、状态、用量、成本、路由尝试，以及后端启用时保存的请求和响应内容。
